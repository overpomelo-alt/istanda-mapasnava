# Istanda Mapasnava — 北極星（North Star）

> 每次猶豫某個功能要不要做時，回來讀這份。
> 如果這個功能會讓 6/15 那天的瞬間、或 6/22 那週的使用，變得更動人 → 做。
> 不會 → 刪。
>
> 比任何技術決策都重要。

---

## 🎯 一句話定位

**Istanda Mapasnava 不是錄音工具，是 87 人家族的私密 Instagram。**

它的成功不是「6/15 那天有人哭」——是「6/22 早上，媽媽不用人提醒就自己打開」。

---

## 第一層：6/15 那天的瞬間

**場景**：家族聚會的傍晚。

```
你拿出手機，打開 App。
深色 hero、山脈、太陽。底下 87 個家人。

「我們家族的 App 上線了。」

弟弟搶過手機，滑到自己那一頁，
按金色麥克風錄了一句話給堂弟笑死。

媽媽在旁邊看著，
「我也可以嗎？」
你按到 Cina Umav 的頁面，把手機遞過去。
她錄一段：「Madadaingaz tu bunun...」

按上傳。

3 秒後 LINE 家族群組跳通知——
小Iman（在台北唸書）：「啊啊啊我哭了 ❤️」
堂哥（在花蓮）：「阿嬤的聲音好溫柔」
小朋友群一個個按愛心。

媽媽看著手機，
「他們聽到了？」
「對，他們聽到了。」
```

**這個瞬間的關鍵**：

不是「媽媽的聲音被保存」，
是「**家族即時的化學反應**」——
是 87 個人在同一個瞬間連起來。

---

## 第二層：6/22 那一週的場景

6/15 之後一週，更難、也更重要。

**場景 A：清晨 6:30**

媽媽自己拿起手機。打開 App。看到孫子昨晚錄的「跟阿嬤學說 Madadaingaz」。聽完按愛心。然後自己又錄了一段。

**場景 B：傍晚 LINE 群組**

小朋友 A：「誰幫我錄一句『我好餓』的布農族語」
堂姊：「⬆️ 我剛在 App 錄了」
3 個人按愛心、4 則留言。

**場景 C：深夜**

爸爸睡不著。打開 App 聽媽媽前幾天錄的那段。聽兩遍。沒留言、沒按讚，但打開了。

---

**衡量「成功」的標準**：

不是 6/15 那天有多少人錄音。
是 **6/22 那天打開 App 的人數 ≥ 6/15 那天**。

如果一週後沒人打開，就是失敗——不管功能多齊全。

---

## 第三層：決策對照清單

遇到「這個功能要不要做？」的時候，問自己：

### ✅ 6/15 必做（沒這些就不算上線）

**核心錄音**
- 大金色麥克風按鈕、點擊切換 → **讓爺奶不怕第一次**
- 試聽 / 重錄 / 上傳 → **讓爺奶知道講錯沒關係**
- 個人頁面族名、暱稱、頭像 → **讓每個人知道這就是「他」的位置**
- 過去錄音清單 → **讓每個人看到自己/家人累積了什麼**

**社群互動（讓 87 人連起來）**
- 按讚 ❤️ → **不用打字、3 歲到 90 歲都會用**
- 留言 💬 → **年輕人才會回訪的引擎**
- LINE 一鍵分享 → **把 App 接到家族真實的溝通場域**

**回訪引擎**
- PWA 安裝（加到主畫面） → **沒有桌面 icon，第二天就忘了**
- 首頁動態 Feed 即時更新 → **讓打開的人馬上看到新東西**

### 🟡 加分（時間有就做）

- 真實頭像照片（取代縮寫圓圈）
- 暱稱顯示（小Iman、大Iman 區分）
- 錄音中振動回饋

### ❌ 拒絕做（會破壞瞬間）

- **登入頁** → 媽媽會在這裡放棄、小朋友懶得用
- **太多按鈕** → 長輩會困惑、年輕人覺得 low
- **嚇人的錯誤訊息** → 媽媽會以為自己弄壞了
- **「請輸入標題」這種要求打字的** → 一個輸入框就足以勸退長輩
- **跳轉到外部 App 播影片** → 跳出去就回不來
- **任何「要先教學才會用」的設計** → 不教就會 = 唯一標準

### 🚫 6/15 後再做（不是必做、但有計畫）

- 影片上傳（Google Drive 嵌入）
- 跟著學功能（年輕人跟著錄回傳）
- 管理員頁面
- 族語本

## 視覺與聲音的考量

### 視覺
- 深色 hero + 金色麥克風（#c8a96e）
- 山脈圖騰：告訴大家這是「我們的東西」
- 布農族名（Cina Umav，不是「媽媽」「長輩」）：尊重 + 文化記號

### 聲音
- 錄音中倒數計時：清楚但不焦慮
- 上傳完成的「☁ 上傳成功」：讓人知道「成功了」
- 沒有任何刺耳的提示音

### 觸覺
- 按鈕大、易按
- 點下去有縮放回饋
- 振動回饋（如果做得到）

---

## 衡量成功的三個問題

每週問自己：

1. **6/22 那天打開 App 的人數 ≥ 6/15 那天嗎？**
   - 是 → App 活著
   - 否 → 想想為什麼沒人回來

2. **過去 7 天有沒有人錄了第二段以上？**
   - 是 → 這 App 對他有意義
   - 否 → 第一次錄完就沒回來，是入口問題還是內容問題？

3. **LINE 家族群組這週有出現 App 連結嗎？**
   - 是 → 它接到了真實的家族溝通
   - 否 → 它還只是「另一個 App」、不是「我們的東西」

## 寫給未來的自己

教師手帳是給自己用的工具，可以慢慢迭代。
布農族 App 是給 87 個家人、給布農族的傳承——**第一印象決定一切**。

如果 6/15 上線那天 bug 多但好玩 → 成功。
如果 6/15 完美但無聊 → 失敗。
如果 6/15 沒做完但 6/22 上線時 87 人都嗨起來 → 也成功。

**做最重要的、砍其他的。**

---

## 給每次討論功能時的提醒

**不要問「這個功能酷不酷」。**
**要問「這個功能會讓更多家人在 6/22 那天打開 App 嗎？」**

- 漂亮的動畫效果 → 不會 → 砍
- 真實的家人頭像照片 → 會（更有歸屬感）→ 做
- 複雜的權限管理 → 不會 → 砍
- 留言通知 → 會（被留言的人會想回看）→ 做
